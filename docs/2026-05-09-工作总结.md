# 桌喵（zhuomiao）— 2026-05-09 工作总结

---

## 📋 摘要

今日对桌喵桌面宠物应用进行了**5项关键缺陷修复**和**1项新功能开发**，核心改动是将数据存储从JSON文件迁移到MySQL数据库，并采用乐观更新+即时持久化+失败回滚策略彻底解决任务数据"复活"Bug。同时修复了活动图表截断、任务完成无视觉反馈、面板窗口尺寸不足等UI问题，新增"与桌喵聊天"功能支持自然语言管理任务。最后根据用户反馈修正了降级模式命名（`memory`→`file`），确保MySQL不可用时降级到本地JSON文件存储而非内存。**Release v1.0.3 编译打包通过。**

## 🔴 修复的Bug

### P0-1: 任务数据持久化问题（数据复活）

| 项目 | 详情 |
|------|------|
| **现象** | 用户删除任务后关闭面板再打开，被删除的任务自动复活 |
| **根因** | 使用JSON全量序列化+定时保存，面板重开时从磁盘加载旧数据覆盖内存 |
| **修复方案** | 集成MySQL数据库（sqlx异步驱动），每个增删改操作即时持久化到DB，面板加载时从DB读取权威数据，JSON文件仅作为降级兜底 |
| **关键文件** | `src-tauri/src/db/mod.rs`、`src-tauri/src/commands/task.rs`、`src/lib/services/persistence.ts` |

### P0-2: 活动图表窗口内容截断

| 项目 | 详情 |
|------|------|
| **现象** | 图表内容超出容器，被截断无法查看 |
| **修复方案** | 容器`max-height`从500px增至600px，内部改为`flex column`布局，活动明细列表独立滚动区域200px |
| **关键文件** | `src/routes/+page.svelte`、`src/lib/components/ActivityChart.svelte` |

### P0-3: 任务完成检测后无视觉反馈

| 项目 | 详情 |
|------|------|
| **现象** | AI检测到任务完成条件满足，但任务未被勾选标记 |
| **修复方案** | `confirmTaskCompletion()`改用即时持久化API（`persistToggleTask`），确认后立即写入DB并显示成功气泡，失败时显示"保存失败"提示 |
| **关键文件** | `src/routes/+page.svelte:286` |

### P1: 面板窗口尺寸不足

| 项目 | 详情 |
|------|------|
| **现象** | "清除已完成"按钮在默认窗口尺寸下不完全可见 |
| **修复方案** | 面板高度560→640px，TaskPanel布局改为`flex column`，任务列表可滚动，底部按钮`flex-shrink:0`始终可见 |
| **关键文件** | `src-tauri/tauri.conf.json`、`src/routes/+page.svelte`、`src/lib/components/TaskPanel.svelte` |

## 🏗️ 技术架构变更

### Rust后端

| 变更 | 说明 |
|------|------|
| **新增依赖** | `sqlx 0.8`（MySQL异步驱动）、`dotenvy 0.15`（.env配置）、`tokio 1`（异步运行时）、`chrono 0.4`（时间处理） |
| **数据库模块** | `src-tauri/src/db/mod.rs` — `DbState`连接池管理，`file_mode()`降级，`run_migrations()`自动建表 |
| **数据模型** | `src-tauri/src/models.rs` — `DbTask`、`TaskInput`、`TaskPatch`、`DbActivityRecord`、`DbMonitorRule`、`DbAiConfig`、`DbStatusInfo`、`MigrationReport` |
| **CRUD命令** | `src-tauri/src/commands/` — task(6命令)、activity(3命令)、rule(2命令)、config(4命令)、migration(1命令) |
| **启动流程** | `lib.rs` — 读取`.env`→尝试MySQL连接→成功则运行迁移→失败则降级为文件模式→注入Tauri State |
| **旧命令保留** | `save_app_data`/`load_app_data`保留，作为文件模式的读写通道 |

### 前端

| 变更 | 说明 |
|------|------|
| **持久化服务重构** | `persistence.ts` — 新增`createTask`/`removeTask`/`updateTask`/`toggleTask`/`clearCompleted`/`loadAllFromDB`，乐观更新+即时DB写入+失败回滚到JSON |
| **类型扩展** | `types/index.ts` — 新增`ChatMessage`、`TaskAction`、`ChatResponse`、`DbStatusInfo`、`MigrationReport`、`TaskChatContext` |

### 数据库表结构

```sql
-- 任务表
CREATE TABLE tasks (
    id VARCHAR(36) PRIMARY KEY,
    title VARCHAR(100) NOT NULL,
    category VARCHAR(20) NOT NULL DEFAULT '学习',
    priority VARCHAR(10) NOT NULL DEFAULT 'medium',
    due_date VARCHAR(20),
    completed TINYINT(1) NOT NULL DEFAULT 0,
    created_at VARCHAR(30) NOT NULL,
    completion_hint VARCHAR(200),
    completion_method VARCHAR(20)
);

-- 活动记录表
CREATE TABLE activity_records (
    id VARCHAR(50) PRIMARY KEY,
    timestamp VARCHAR(30) NOT NULL,
    window_title VARCHAR(300) NOT NULL,
    process_name VARCHAR(100) NOT NULL,
    classification VARCHAR(20) NOT NULL DEFAULT 'productive',
    classification_source VARCHAR(20) NOT NULL DEFAULT 'rule_based',
    activity_type VARCHAR(50),
    ai_comment VARCHAR(500),
    task_id VARCHAR(36)
);

-- 监控规则表
CREATE TABLE monitor_rules (
    id VARCHAR(36) PRIMARY KEY,
    pattern VARCHAR(200) NOT NULL,
    rule_type VARCHAR(20) NOT NULL DEFAULT 'url',
    is_blacklist TINYINT(1) NOT NULL DEFAULT 1,
    message VARCHAR(200) NOT NULL DEFAULT ''
);

-- AI配置表
CREATE TABLE ai_config (
    id INT PRIMARY KEY DEFAULT 1,
    provider VARCHAR(50) NOT NULL DEFAULT 'openai',
    endpoint VARCHAR(200) NOT NULL,
    api_key VARCHAR(200) NOT NULL DEFAULT '',
    model VARCHAR(50) NOT NULL DEFAULT 'gpt-4o-mini',
    system_prompt TEXT
);
```

## 🔧 配置说明

使用前需在 `src-tauri/.env` 中配置MySQL连接：

```env
DATABASE_URL=mysql://用户名:密码@localhost:3306/zhuomiao
DB_POOL_MIN=2
DB_POOL_MAX=5
```

- **MySQL可用**：数据存储到MySQL，即时持久化
- **MySQL不可用**：自动降级为本地JSON文件存储（使用设置中配置的数据目录），数据同样持久化到硬盘

## 💡 用户反馈修正

> 用户指出："用内存存数据太傻屌了，如果MySQL配置失败了就存本地，设置里已经指定好了数据存放的位置"

**修正内容**：
- 将降级模式从 `"memory"` 重命名为 `"file"`
- `DbState::memory_mode()` → `DbState::file_mode()`
- MySQL不可用时数据写入本地JSON文件（使用设置中配置的数据目录），而非仅存内存
- 前端降级提示和日志信息均更新为"本地文件存储模式"
